
What is Syntax Analysis?
------------------------

Syntax analysis is the **second phase of a compiler**.It checks whether the sequence of tokens generated by the lexical analyzer **follows the grammatical rules** of the programming language.

### In Simple Words:

*   **Lexical Analysis** → “Are the words correct?”
    
*   **Syntax Analysis** → “Is the sentence correct?”
   
### Real-Life Example:

Sentence:

> “I eating food”

Words are correct, but grammar is wrong.➡️ Syntax analyzer reports an error.

ROLE OF A PARSER
================

What is a Parser?
-----------------

A **parser** is a program that:

*   Takes tokens from the lexical analyzer
    
*   Checks syntax using grammar rules
    
*   Builds a **parse tree**
    

### Input to Parser:
Tokens → id + id * id   `

### Output:

✔ Syntax Valid / ❌ Syntax Error✔ Parse Tree

Functions of a Parser
---------------------

1.  Verifies grammatical structure
    
2.  Reports syntax errors
    
3.  Builds parse tree
    
4.  Helps semantic analysis
    

### Real-Life Example:

A school teacher checks:

*   Subject–Verb agreement
    
*   Sentence structureParser does the same for programs.
    

CONTEXT-FREE GRAMMARS (CFG)
===========================

What is a Grammar?
------------------

A grammar is a **set of rules** that describe how valid statements of a language are formed.

Context-Free Grammar (CFG)
--------------------------

A grammar is called **context-free** if all its production rules are of the form:
 A → α   `

Where:

*   A = Non-terminal
    
*   α = Terminals and/or non-terminals
    

Components of CFG
-----------------

A CFG has **four parts**:

1.  **Terminals** – Actual symbols (id, +, \*)
    
2.  **Non-terminals** – Variables (E, T, F)
    
3.  **Start Symbol** – First non-terminal
    
4.  **Productions** – Grammar rules
    

Example of CFG
--------------
E → E + T | T  T → T * F | F  F → (E) | id   `

### Meaning:

*   Expressions contain terms
    
*   Terms contain factors
    

Real-Life Example:
------------------

English Grammar:
 Sentence → Noun Phrase + Verb Phrase   `

Programming Grammar:
Expression → Expression + Term   `

USE OF CFG IN PROGRAMMING LANGUAGES
===================================

CFG is used to:

*   Define syntax of languages (C, Java)
    
*   Build parsers
    
*   Detect syntax errors
    

### Example:

C language if statement:
if (condition) statement   `

CFG Rule:
stmt → if ( expr ) stmt   `

TECHNIQUES FOR WRITING GRAMMARS
===============================

1️⃣ Removal of Left Recursion
-----------------------------

### What is Left Recursion?

A grammar is **left recursive** if a non-terminal appears on the **leftmost side** of its production.

Example:
 A → Aα | β   `

### Why Remove Left Recursion?

*   Top-down parsers (LL parsers) **cannot handle** left recursion
    
*   Causes infinite loop
    

### Example of Left Recursion
E → E + T | T   `

### Removal of Left Recursion

Step 1: Identify α and β
E → T E'  E' → + T E' | ε   `

✔ Left recursion removed

### Real-Life Example:

Circular instruction:

> “Before doing A, do A”

This creates confusion, same as left recursion.

2️⃣ Left Factoring
------------------

### What is Left Factoring?

Used when multiple productions have **common prefixes**.

Example:
 A → αβ1 | αβ2   `

### Left Factored Form:
A → α A'  A' → β1 | β2   `

### Example:

Before:
stmt → if expr then stmt else stmt  | if expr then stmt   `

After Left Factoring:
  stmt → if expr then stmt stmt'  stmt' → else stmt | ε   `

NON-CONTEXT-FREE CONSTRUCTS IN PROGRAMMING LANGUAGES
====================================================

Some language features **cannot be expressed using CFG alone**.

Examples:
---------

### 1️⃣ Variable Declaration Checking
int x;  x = 5;   `

CFG cannot check whether x is declared or not.

### 2️⃣ Type Checking
int a;  a = "hello"; ❌   `

CFG cannot detect type mismatch.

### 3️⃣ Matching Number of Parameters

Function call vs function definition

### How Are They Handled?

*   Semantic Analysis
    
*   Symbol Table
    
*   Context-Sensitive Rules
    

PARSE TREES
===========

What is a Parse Tree?
---------------------

A parse tree is a **hierarchical tree structure** that shows how a string is derived using grammar rules.

Example Grammar:
----------------
E → E + T | T  T → id   `

Input:
id + id   `

### Parse Tree:
`E         /|\        E + T        |   |        T  id        |       id`

Importance of Parse Tree
------------------------

*   Shows syntactic structure
    
*   Helps semantic analysis
    
*   Used in code generation
    

AMBIGUITY IN GRAMMAR
====================

What is Ambiguous Grammar?
--------------------------

A grammar is **ambiguous** if a string has **more than one parse tree**.

Example of Ambiguous Grammar
----------------------------
   E → E + E | E * E | id   `

Input:
 id + id * id   `

Two interpretations:

1.  id + (id \* id)
    
2.  (id + id) \* id
    

### Why Ambiguity is a Problem?

*   Confusing meaning
    
*   Wrong evaluation order
    

### Removing Ambiguity

Use operator precedence grammar:
E → E + T | T  T → T * F | F  F → id   `

EXAMPLES OF PROGRAMMING LANGUAGE GRAMMARS
=========================================

### Assignment Statement:
 stmt → id = expr ;   `

### If Statement:
 stmt → if ( expr ) stmt else stmt   `

### While Loop:
 stmt → while ( expr ) stmt   `
